Esta aplicación ha sido desarrollada como parte de una prueba técnica, tomando como base la plantilla de inicio de ASP.NET Boilerplate. Se ha personalizado y extendido para cumplir con los requisitos funcionales y técnicos de un módulo de solicitudes de crédito en línea.

Módulo de Solicitudes de Crédito

El sistema cuenta con dos tipos de usuario:

  • Solicitante: puede registrarse, iniciar sesión, crear nuevas solicitudes y consultar su historial personal.
  • Analista: tiene acceso a todas las solicitudes, con capacidad para filtrarlas por estado y aprobar o rechazar según corresponda.

Por defecto, al levantar el sistema por primera vez, se crea automáticamente el rol de analista. Un usuario administrador puede asignar este rol a cualquier otro usuario desde la pantalla de administración de usuarios.

Una vez asignado, el usuario verá en el menú lateral la opción “Solicitudes (Analista)”, desde donde podrá acceder a todas las solicitudes registradas y gestionar su estado.

Las solicitudes tienen un estado inicial determinado automáticamente por el sistema según el ingreso mensual:

  • Si el ingreso mensual es igual o mayor a $1500 → el estado inicial será Aprobado.
  • En caso contrario, el estado inicial será Pendiente.

Como medida de integridad, el sistema no permite que un analista apruebe sus propias solicitudes (se valida por usuario).

Base tecnológica

  • Frontend: Angular
  • Backend: ASP.NET Core (Boilerplate)
  • Autenticación: JWT
  • Auditoría automática y validación de roles integrada
